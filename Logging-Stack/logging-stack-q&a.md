# Logging Stack Interview Questions & Answers (DevOps - 2 to 5 Years)

---

## 1. What is logging?
Logging is the process of recording events from applications and systems.

---

## 2. Why logging is important?

- Debug issues  
- Monitor systems  
- Track errors  
- Audit activities  

---

## 3. What is a logging stack?
A logging stack is a combination of tools used to collect, store, and visualize logs.

---

## 4. What is ELK stack?
ELK stands for:
- Elasticsearch → stores logs  
- Logstash → processes logs  
- Kibana → visualizes logs  

---

## 5. What is Elasticsearch?
A search and analytics engine used to store and query logs.

---

## 6. What is Logstash?
A tool used to collect and process logs before sending to storage.

---

## 7. What is Kibana?
A visualization tool used to search and analyze logs.

---

## 8. What is Fluentd / Fluent Bit?
Lightweight log collectors used to forward logs.

---

## 9. What is Loki?
A log aggregation system used with Grafana.

---

## 10. Difference between logs and metrics?

- Logs → detailed events  
- Metrics → numerical values  

---

## 11. What is centralized logging?
Collecting logs from multiple systems into one place.

---

## 12. What is log rotation?
Managing log file size by rotating old logs.

---

## 13. What is log level?

- INFO  
- WARN  
- ERROR  
- DEBUG  

---

## 14. What is structured logging?
Logs in structured format like JSON for easy parsing.

---

## 15. What is log parsing?
Extracting useful data from logs.

---

## 16. What is log retention?
How long logs are stored.

---

## 17. What is alerting in logging?
Trigger alerts based on log patterns.

---

## 18. How logging is used in DevOps?

- Debugging  
- Monitoring  
- Auditing  
- Security  

---

## 19. What is log ingestion?
Process of collecting logs from sources.

---

## 20. Example flow

Application → Log collector → Storage → Visualization
