# DevOps Security Interview Questions & Answers (DevOps - 2 to 5 Years)

---

## 1. What is DevOps Security (DevSecOps)?
DevSecOps is the practice of integrating security into every stage of the DevOps lifecycle.

---

## 2. Why security is important in DevOps?
- Prevent data breaches  
- Protect applications  
- Ensure compliance  
- Avoid production issues  

---

## 3. What is IAM (Identity and Access Management)?
IAM controls who can access resources and what actions they can perform.

---

## 4. What is least privilege principle?
Users should only have minimum permissions required.

---

## 5. What is authentication vs authorization?

- Authentication → Who you are  
- Authorization → What you can do  

---

## 6. What is secrets management?
Securely storing sensitive data like passwords, API keys.

---

## 7. What tools are used for secrets?

- Vault  
- AWS Secrets Manager  
- Kubernetes Secrets  

---

## 8. What is SSL/TLS?
Protocols used to encrypt communication between systems.

---

## 9. What is HTTPS?
Secure version of HTTP using SSL/TLS.

---

## 10. What is encryption?

- Data is converted into secure format  
- Types:
  - At rest  
  - In transit  

---

## 11. What is container security?
Securing Docker images and containers.

---

## 12. What is image scanning?
Checking Docker images for vulnerabilities.

---

## 13. What is network security?
Protecting communication between systems.

---

## 14. What is firewall?
Controls incoming and outgoing traffic.

---

## 15. What is security group (cloud)?
Virtual firewall controlling traffic to instances.

---

## 16. What is vulnerability scanning?
Identifying security weaknesses.

---

## 17. What is patch management?
Updating systems to fix vulnerabilities.

---

## 18. What is RBAC?
Role-Based Access Control to manage permissions.

---

## 19. What is secure CI/CD pipeline?

- Use secrets securely  
- Scan code  
- Restrict access  

---

## 20. How security is implemented in DevOps?

- Secure code  
- Secure infrastructure  
- Monitor threats  
- Automate security checks  

---

## Bonus Example

Secure pipeline:
Code → Scan → Build → Scan → Deploy
