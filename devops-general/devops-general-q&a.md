# General DevOps Interview Questions & Answers (2–5 Years)

---

## 1. What is DevOps?
DevOps is a culture and set of practices that combines development and operations teams.  
The goal is to deliver software faster, more reliably, and with better quality using automation and collaboration.

---

## 2. Why DevOps is important?
DevOps helps:
- Faster delivery of applications  
- Reduced manual work  
- Better collaboration  
- Improved system stability  

---

## 3. What is CI/CD?
CI/CD stands for:
- CI (Continuous Integration) → Code is automatically built and tested  
- CD (Continuous Deployment/Delivery) → Code is automatically deployed  

---

## 4. What is Continuous Integration?
Developers frequently push code to repository, and automated builds and tests run to detect issues early.

---

## 5. What is Continuous Deployment?
Automatically deploying code changes to production without manual approval.

---

## 6. What is Continuous Delivery?
Code is ready for deployment, but requires manual approval before production.

---

## 7. What is Infrastructure as Code (IaC)?
Managing infrastructure using code instead of manual setup.  
Tools: Terraform, CloudFormation.

---

## 8. What is Docker?
Docker is a containerization tool used to package applications and dependencies into containers.

---

## 9. What is Kubernetes?
Kubernetes is a container orchestration tool used to manage and scale containers.

---

## 10. What is a container?
A container is a lightweight environment that runs an application with its dependencies.

---

## 11. What is virtualization?
Virtualization creates virtual machines that run on physical hardware.

---

## 12. Difference between VM and container?
- VM → heavy, full OS  
- Container → lightweight, shares OS  

---

## 13. What is cloud computing?
Cloud computing provides resources like servers, storage, and networking over the internet.

---

## 14. What are cloud providers?
Examples:
- AWS  
- Azure  
- Google Cloud  

---

## 15. What is monitoring?
Monitoring tracks system performance and health using metrics and logs.

---

## 16. What is Prometheus?
Prometheus is a monitoring tool used to collect metrics.

---

## 17. What is Grafana?
Grafana is a visualization tool used to create dashboards.

---

## 18. What is networking?
Networking connects systems and allows communication between them.

---

## 19. What is a load balancer?
Distributes traffic across multiple servers to improve performance.

---

## 20. What is version control?
Version control tracks changes in code.  
Example: Git

---

## 21. What is Jenkins?
Jenkins is a CI/CD tool used to automate build, test, and deployment.

---

## 22. What is a pipeline?
A pipeline is a series of automated steps (build, test, deploy).

---

## 23. What is automation in DevOps?
Automation reduces manual work by using scripts and tools.

---

## 24. What is logging?
Logging records system events for debugging and monitoring.

---

## 25. What is scalability?
Ability to handle increasing load by adding resources.

---

## Bonus Tip

DevOps = Automation + Collaboration + Monitoring + Continuous Delivery
