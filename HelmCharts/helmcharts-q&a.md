# Helm Interview Questions & Answers (DevOps - 2 to 5 Years)

---

## 1. What is Helm?
Helm is a package manager for Kubernetes used to deploy and manage applications.

---

## 2. Why Helm is used?

- Simplifies deployments  
- Reusable configurations  
- Version control for deployments  

---

## 3. What is a Helm chart?
A Helm chart is a collection of files used to define a Kubernetes application.

---

## 4. What are components of a Helm chart?

- Chart.yaml  
- values.yaml  
- templates/  
- charts/  

---

## 5. What is values.yaml?
File used to define default values for templates.

---

## 6. What is templating in Helm?
Helm uses templates to generate Kubernetes YAML dynamically.

---

## 7. What is a release in Helm?
A running instance of a Helm chart.

---

## 8. What is helm install?
Installs a chart and creates a release.

---

## 9. What is helm upgrade?
Updates an existing release.

---

## 10. What is helm rollback?
Reverts to a previous release version.

---

## 11. What is helm uninstall?
Deletes a release.

---

## 12. What is helm repo?
A repository where charts are stored.

---

## 13. What is dependency in Helm?
Charts can depend on other charts.

---

## 14. What is helm lint?
Checks chart for errors.

---

## 15. What is helm template?
Generates YAML without deploying.

---

## 16. What is helm upgrade --install?
Installs if not present, otherwise upgrades.

---

## 17. What is chart versioning?
Tracking versions of charts.

---

## 18. What is values override?
Passing custom values at runtime.

---

## 19. How Helm is used in DevOps?

- Automate Kubernetes deployments  
- Manage configurations  
- Enable CI/CD integration  

---

## 20. Example workflow

Chart → Values → Templates → Deploy → Release
