# DNS & Networking Interview Questions & Answers (DevOps - 2 to 5 Years)

---

## 1. What is networking?
Networking is the communication between systems over a network.

---

## 2. What is DNS?
DNS (Domain Name System) converts domain names into IP addresses.

---

## 3. What happens when you enter a URL in browser?

- DNS lookup  
- Get IP address  
- Request sent to server  
- Server responds  

---

## 4. What is IP address?
Unique identifier for a device on network.

---

## 5. What is public vs private IP?

- Public → accessible from internet  
- Private → internal network  

---

## 6. What is subnet?
A smaller network inside a larger network.

---

## 7. What is gateway?
Entry/exit point for network traffic.

---

## 8. What is port?
Logical endpoint for communication.  
Example: 80 (HTTP), 443 (HTTPS)

---

## 9. What is HTTP vs HTTPS?

- HTTP → not secure  
- HTTPS → encrypted  

---

## 10. What is TCP vs UDP?

- TCP → reliable, connection-based  
- UDP → fast, connectionless  

---

## 11. What is load balancer?
Distributes traffic across multiple servers.

---

## 12. What is L4 vs L7 load balancer?

- L4 → based on IP/port  
- L7 → based on HTTP/URL  

---

## 13. What is reverse proxy?
Server that forwards client requests to backend servers.

---

## 14. What is firewall?
Controls incoming/outgoing traffic.

---

## 15. What is NAT?
Network Address Translation maps private IP to public IP.

---

## 16. What is DNS record types?

- A → IP mapping  
- CNAME → alias  
- MX → mail  
- TXT → verification  

---

## 17. What is latency?
Time taken for data transfer.

---

## 18. What is bandwidth?
Amount of data that can be transferred.

---

## 19. What is CDN?
Content Delivery Network caches content closer to users.

---

## 20. Example flow

User → DNS → Load Balancer → Server → Response
